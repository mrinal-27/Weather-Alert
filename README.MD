# WeatherGuard Admin

WeatherGuard is a secure, invite-only weather alert platform designed to demonstrate the integration of multiple systems within a maintainable and scalable architecture. The application combines Google OAuth authentication, an administrator approval workflow, MongoDB data storage, Telegram bot integration, and automated weather notifications into a single end-to-end solution.

The platform allows users to request access through Google Sign-In. Once authenticated, a user account is created in the database with a default status of **Pending**. Administrators can review incoming requests through a dedicated React-based dashboard and manually approve or reject users. This approval process ensures that only authorized users are able to receive notifications from the system.

The backend is built using NestJS and follows a modular architecture that separates authentication, user management, administration, Telegram integration, weather services, and scheduling into independent modules. This separation improves maintainability and makes the application easier to extend in the future. MongoDB is used for persistent storage, while Mongoose provides schema validation and data modeling.

## Database Schema

The application uses a single User collection to manage authentication, approval status, and Telegram integration. Each user record contains personal information obtained through Google OAuth, approval status information, and Telegram linking details.

Fields stored for each user include the user's name, email address, OAuth provider information, approval status, Telegram chat identifier, temporary Telegram verification code, administrator flag, and automatic timestamps.

## Data Flow

The application follows a controlled approval workflow. When a user signs in using Google OAuth, the backend checks whether the user already exists. If the user does not exist, a new record is created with a status of **Pending**. The user is then redirected to the status page where they can monitor the progress of their request.

Administrators access the dashboard and review pending requests. When a user is approved, their status changes from **Pending** to **Approved**. The backend immediately sends a Telegram notification informing the user that access has been granted. Rejected users are updated with a **Rejected** status and are prevented from accessing the Telegram integration process.

After approval, users can connect their Telegram account through a secure verification process. A temporary linking code is generated by the backend and presented to the user. The user sends this code to the Telegram bot, allowing the system to associate the Telegram chat with the corresponding account stored in MongoDB.

Only users whose status is marked as **Approved** and whose Telegram account has been linked are eligible to receive weather alerts. The scheduler periodically retrieves weather information from the weather service and distributes notifications through the Telegram bot. This ensures that pending or rejected users never receive weather updates.

## Architecture

The frontend is developed using React, TypeScript, Tailwind CSS, and React Router. The backend is developed using NestJS, Passport.js, MongoDB, Mongoose, and scheduled background tasks. Communication between the frontend and backend occurs through REST APIs.

The project is divided into the following modules:

* Auth Module – Handles Google OAuth authentication.
* Users Module – Manages user records and account status.
* Admin Module – Provides approval and rejection functionality.
* Telegram Module – Sends messages and manages Telegram integration.
* Weather Module – Retrieves weather information from external APIs.
* Scheduler Module – Executes automated weather notification jobs.

## Setup Instructions

Clone the repository and install dependencies for both the backend and frontend applications.

Backend:

```bash
cd api
npm install
npm run start:dev
```

Frontend:

```bash
cd admin
npm install
npm run dev
```

Create a `.env` file inside the backend directory and configure the MongoDB connection string, Google OAuth credentials, Telegram bot token, JWT secret, and OpenWeather API key.
